# MyGym / repEtition Project Brief

## Snapshot
- Project: MyGym / repEtition
- Website: https://rep-etition.co.za/
- Branch for active work: `dev`
- Status: Active build + go-to-market preparation

## Current intent
China wrote this app and wants to:
- improve and harden the codebase
- get the app market-ready
- prepare it for go-to-market
- build a repeatable operating protocol for product, code, and strategy work

## Working product framing
Current website positioning suggests the app is:
- an AI-powered workout tracker
- built for serious lifters
- local-first
- focused on speed, clarity, and low fluff

Website messaging currently emphasizes:
- AI Coach workout generation
- blank workout mode
- custom exercise library
- per-exercise history
- local-first storage
- export / backup
- offline-capable experience

## Project goals
This project currently has two main tracks:

### 1) Product / go-to-market
- clarify positioning
- improve onboarding and external legibility
- decide launch path
- shape messaging and acquisition approach

### 2) App / engineering
- review current implementation quality
- identify technical debt and risk areas
- improve architecture, polish, reliability, and readiness
- implement prioritized features and fixes on `dev`

## Known concerns
- China uses the app daily and is concerned it may make more sense to him than to new users.
- Future analysis should focus on:
  - external clarity
  - positioning
  - onboarding
  - whether the product value is legible to other users
