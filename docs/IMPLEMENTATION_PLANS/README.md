# Implementation Plans

This folder holds temporary working plans for significant MyGym changes.

Use this folder when a change is large enough to benefit from explicit structure before coding.

Typical contents:
- problem summary
- desired outcome
- approach
- files/modules likely affected
- risks and edge cases
- validation checklist

Do not create a plan here for every small change.

When a plan is complete:
- keep it if it remains useful as historical context
- otherwise consolidate durable decisions into `../DECISIONS.md` and let the plan remain a finished artifact
