# MyGym Docs Map

This folder is the structured project home for MyGym.

## Active workflow
Start with:
- `DEV_WORKFLOW.md`

Core principle:
- one inbox
- one planning layer
- one project home

For MyGym:
- Apple Notes = raw capture inbox
- Tjop = planning / routing layer
- repo docs = structured project truth
- codebase = implementation truth
- Obsidian = deprecated for active developer workflow

## Core docs
Use these most often:
- `PROJECT_BRIEF.md` -> stable project framing
- `DEV_WORKFLOW.md` -> how development work flows
- `INTAKE_PROTOCOL.md` -> how raw MyGym inputs get classified and routed
- `BACKLOG.md` -> actionable queued work
- `DECISIONS.md` -> durable technical / product decisions

## Supporting docs
Use when needed:
- `CODE_READINESS.md` -> technical health and release-readiness view
- `GTM.md` -> positioning / launch / market thinking
- `FEATURE_INVENTORY.md` -> broad feature bank and structured feature understanding

## Working docs
These are active-plan or validation artifacts, not the main operating system:
- `IMPLEMENTATION_PLANS/` -> significant implementation plans
- `CHECKLISTS/` -> manual validation checklists

## Rules of thumb
- raw ideas go to Apple Notes first unless immediate discussion is easier
- not every idea becomes a doc
- not every change needs an implementation plan
- only durable decisions belong in `DECISIONS.md`
- only actionable queued work belongs in `BACKLOG.md`
- avoid duplicating the same active item across multiple docs
